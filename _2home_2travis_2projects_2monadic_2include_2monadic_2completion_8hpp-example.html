Perform the next step of the process if the value is delivered in failure. The func is only called in the failure case (see map for the success case).

Template Parameters
FuncR (*)(std::exception_ptr&&); R can be void
Parameters
funcThis function takes in an std::exception_ptr and returns some result to continue the chain with. The result of this function will be delivered to the completion_promise that backs the completion returned from this function. If the result of func is non-void, it must be convertable to T – the resulting completion is the common type of T and R. If the result of func is void, then T must also be void.
Returns
a completion that will be completed with the result of func.
completion_promise<int> p;
completion<void> c = p.get_completion()
.map([] (int x) -> int { assert(false && "Will never be called"); return 0; })
.recover([] (std::exception_ptr ex_ptr)
{
try
{
std::rethrow_exception(ex_ptr);
}
catch (const std::exception& ex)
{
return std::strlen(ex.what());
}
})
// This will be called because we have "recovered":
.map([] (int x) { return x * 2; })
.map([] (int x) { std::cout << x << std::endl; });
try
{
throw std::logic_error("Something?");
}
catch (...)
{
p.set_exception(std::current_exception());
}
c.get();
