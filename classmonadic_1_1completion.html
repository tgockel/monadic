Detailed Description

template<typename T>
class monadic::completion< T >

A completion is a monadic version of a std::future.

It can mostly be used as a drop-in replacement for a std::future, with the added benefit of having functions like map and recover.

Disables this completion.

It cannot be un-disabled! This is useful for notifying the thread fulfilling the promise that the receiver no longer cares about the value and is free to abandon processing.

template<typename T >
T monadic::completion< T >::get ( )
inline

Blocks waiting on the result of this completion.

If the completion was completed in success, the value will be returned (on void, nothing is returned). If the completion was completed in failure, the exception will be thrown.

Note
This is not the intended use for a completion and only exists for convenient compatibility with std::future. While the majority of completion functions are implemented using spin locks, this function is expected to block, so it relies on std::promise and std::future, which are slow by comparison.
